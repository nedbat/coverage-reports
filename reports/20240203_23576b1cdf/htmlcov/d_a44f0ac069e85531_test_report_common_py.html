161class ReportWithJinjaTest(CoverageTest): 1abcdefghijklmnopqrstuvwxyzABCTDEFGHIJKLMNOPQRS

162 """Tests of Jinja-like behavior. 

163 

164 Jinja2 compiles a template into Python code, and then runs the Python code 

165 to render the template. But during rendering, it uses the template name 

166 (for example, "template.j2") as the file name, not the Python code file 

167 name. Then during reporting, we will try to parse template.j2 as Python 

168 code. 

169 

170 If the file can be parsed, it's included in the report (as a Python file!). 

171 If it can't be parsed, then it's not included in the report. 

172 

173 These tests confirm that code doesn't raise an exception (as reported in 

174 #1553), and that the current (incorrect) behavior remains stable. Ideally, 

175 good.j2 wouldn't be listed at all, since we can't report on it accurately. 

176 

177 See https://github.com/nedbat/coveragepy/issues/1553 for more detail, and 

178 https://github.com/nedbat/coveragepy/issues/1623 for an issue about this 

179 behavior. 

180 

181 """
